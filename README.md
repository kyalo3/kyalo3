## 🚀 Featured Projects

### 🌱 SustainaShare — Food Donation Management Platform

> *Empowering communities through transparent, trackable food donation and distribution.*

My flagship project — a comprehensive platform that manages the full donation lifecycle from request creation to fulfillment, with role-based dashboards for every stakeholder.

| Repository | Role | Stack |
|------------|------|-------|
| [**430**](https://github.com/kyalo3/430) | Full-stack monorepo (frontend + backend) | React 18, Vite, Tailwind CSS, FastAPI, MongoDB |
| [**430_backend**](https://github.com/kyalo3/430_backend) | Standalone FastAPI REST API | Python, FastAPI, Pydantic, Uvicorn, JWT |
| [**429**](https://github.com/kyalo3/429) | Earlier full-stack iteration | React, Vite, Tailwind, FastAPI, MongoDB |
| [**sustainashare_backend**](https://github.com/kyalo3/sustainashare_backend) | Backend service layer | Python, FastAPI |
| [**sas**](https://github.com/kyalo3/sas) | Extended documentation & architecture | React, FastAPI, MongoDB Atlas |

**Key features:**
- 🔐 JWT-based authentication with role-based access control (Admin · Donor · Recipient · Volunteer)
- 📊 Interactive dashboards with analytics, activity feeds, and system health monitoring
- 🍽️ End-to-end donation lifecycle — request → approval → funding → fulfillment → confirmation
- 📱 Responsive, modern UI built with Tailwind CSS and reusable React components
- 🗄️ MongoDB Atlas for flexible, scalable data storage
- 📐 Documented data flow diagrams (Context DFD, Level 1 DFD) for system architecture

**Tech highlights:** React Router · Formik & Yup · Axios · async FastAPI endpoints · Pydantic models · Docker-ready backend

### 🍎 Food Donation App — Database Architecture

[**food_donation_app**](https://github.com/kyalo3/food_donation_app) · ⭐ 3 stars · HTML / MongoDB

Designed the **MongoDB schema** for a food donation platform, covering six core collections:

| Collection | Purpose |
|------------|---------|
| `Users` | Donor, recipient, and admin profiles with role-based access |
| `Donations` | Item descriptions, quantities, expiry dates, pickup availability |
| `Requests` | Recipient donation requests with approval workflow |
| `Communications` | In-platform messaging between users and administrators |
| `Analytics` | Donation counts, user metrics, and participation rates |
| `Settings` | System configuration and platform settings |

This project laid the data foundation that evolved into the SustainaShare platform.
